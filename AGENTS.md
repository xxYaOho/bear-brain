## Conventional Commits

格式：`<type>(<scope>): <subject>`

- scope 可选
- 首行不超过 50 字符
- 需要补充说明时，空一行后写 body

| Type     | 说明                     |
| -------- | ------------------------ |
| feat     | 新功能                   |
| fix      | 修复 bug                 |
| refactor | 重构                     |
| docs     | 文档                     |
| chore    | 杂项（依赖、配置、工具） |

Breaking Change：type 后加 `!`，footer 写 `BREAKING CHANGE: 说明`

### 提交语言

- (默认)个人项目：中文
- 开放, 协作或国际化项目：英文
